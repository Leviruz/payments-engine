# ADR 0001 - Use Typescript

    Status: Accepted

    Context:
    We need a technology set capable of creating a stable and scalable system that simulates a Fintech Payment Processing System

    Decision:
    Typescript will be used as the project language.

    Consequences:
    - More familiarity with the language I'm current working with
    - Strong type safety to reduce bugs in the logic
    - Scalable, event-driven architecture that suits high-throughput payment processing
